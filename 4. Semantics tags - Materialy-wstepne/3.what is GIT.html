GIT - kontrola wersji

20.11.2019 |20:10

GIT jest systemem kontroli wersji, czyli narzędziem służącym programistom i testerom do śledzenia zmian w kodzie źródłowym. Umożliwia on także jednoczesną pracę nad jednym projektem wielu różnym pracownikom. Oznacza to, że w tym samym czasie kilku programistów “na raz” może dokonywać zmian w kodzie (tworzyć / edytować / usuwać).

Praktycznie każdy profesjonalny projekt programistyczny posiada system kontroli wersji. Jest to niezwykle pomocne narzędzie, które:

  • umożliwia pracę w tym samym czasie na jednym zestawie plików (co jest pomocne, w przypadku, gdy Twój cały team pracuje np. z różnych zakątków świata),
  • mamy całkowitą kontrolę nad kodem - jesteśmy w stanie szybko zidentyfikować problematyczne zmiany oraz przez kogo zostały one wprowadzone,
  • archiwizujemy cały zapis kodu, tak, że w razie potrzeby możemy powrócić do poprzedniej wersji, co oznacza, że mamy nieustanną kontrolę nad wszystkimi zmianami,
  • umożliwia kontakt i wymianę istotnych informacji z repozytoriami zdalnymi.


Gitowy słowniczek

  • Repozytorium - jest to miejsce, w którym przechowywane są pliki podlegające kontroli wersji. Projekt może się składać z wielu repozytoriów. Repozytorium przechowuje też wszystkie zmiany, dzięki czemu można podejrzeć kto zmienił co i kiedy (przechowuje więc historię).
  • Branch - każda z wersji w systemie wersjonowania umieszczona jest w pewnej hierarchii wersji. Hierarchia ta nazywa się Branch’ą. Z dowolnej już istniejącej wersji możemy w każdym momencie stworzyć nową hierarchię, istniejącą równolegle do już istniejącej, czyli nową Branch. W GITcie najczęściej główną Branchę nazywamy Master.
  • Commitowanie - jest to proces, w którym aktualny stan wersjonowanego pliku lub repozytorium (w GITcie tworzymy wersje całego repozytorium) zapisywany jest jako nowa wersja na aktualnej Branchy,
  • Pull - jest to polecenie pobrania oraz integracji aktualnej wersji ze zdalnej lub dowolnej innej Branchy,
  • Mergowanie - proces integrowania (scalania) wszystkich dotychczasowych zmian
  • Pull Request - prośba do właścicieli repozytorium o pobranie lokalnych zmian,
  • GitHub - platforma, która jest serwerem zdalnym (remote) przechowującą Twoje repozytoria z kodem.
